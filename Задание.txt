Задание:
	Проверть работоспособность таблицы при её заполнении, фильтрации, группировке, сортировке (пробовать как по отдельности, так и комбинируя действия).
Описание:
	- создать новый проект DevExpress, все дальнейшие действия будут производится в этом проекте;
	- создать два визуальных компонента GridControl из библиотек DevExpress'а. Один будет "master", другой "slave";
	- создать для master'а DataSet и сгенерировать для него данные;
	- создать для slave'а пустой DataSet. Данные в него будут попадать из master'а: двойной клик по строке master'a должен создавать такую же запись в slave'е;
	- выполнить задание на slave'е.

Суть задачи - локализовать баг, который тяжело найти в реальном приложении - из-за большого объёма кода (~900 классов), зависимеостей,
многопоточности и т.д.
После выполнения данных шагов из реального проекта были перетянуты классы, создающие реальную таблицу и логику её заполнения.
Заполнятся она так же, как в реальном проекте. Выяснилось, что slave таблица "падает" при сортировке по колонке
Count, причина этого - несоответствие типов
